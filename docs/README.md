# docs/

This folder is the source for the public VitePress documentation site.

Keep `docs/` focused on shipped documentation:

- package usage
- product positioning and package boundaries
- public API conventions
- supported versions and generated resource inventory
- contributor notes about how the docs site works

Do not use `docs/` for scratch notes, internal design proposals, or temporary planning documents.

When design work needs to be preserved for future implementation, prefer:

- GitHub issues for scoped follow-up work
- PR descriptions or review threads for change-specific reasoning
- repository guidance such as `AGENTS.md` when future agents need a durable rule
